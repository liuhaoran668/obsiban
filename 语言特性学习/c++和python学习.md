
## 一：不同点：
C++ 是一种静态类型的语言，而 Python 是一个动态类型的语言。

1. 静态类型（C++）
   静态类型的特点是存在编译期。在编译阶段，编译器可以查找类型错误和相关问题，并完成所需类型的确定。变量的类型或函数的行为在编译期间就已经确定了。

2. 动态类型（Python）
   Python 的动态性表现在其运行时特性。变量的类型需要在运行阶段进行确认。

3. C++ 中的动态行为
   在 C++ 中，虚函数的多态表现为一种动态行为。它通过虚函数表（vtable）实现，在运行时通过查询虚函数表来确定具体的函数行为。

需要注意的是，动态行为是存在额外开销的，而静态行为在编译期间就已完成确认。

---

C++ 也是一个具备内存管理能力的语言，可以对内存进行直接操作。相比之下，Python 只能使用 GC（垃圾回收）机制，且其对象通常只能在堆上分配。

而 C++ 的内存管理具有以下特性：
1. 栈内存使用：C++ 可以在栈上分配内存。其直接声明的变量全部都在栈上进行声明和使用。
2. 堆内存申请：通过指针以及 new 和 delete 关键字在堆上申请内存。

由于栈的分配效率天然比堆快，所以 C++ 在栈上申请变量，运行的速度会非常快，在这一方面具有明显优势。在进行系统设计时，应当充分利用 C++ 的内存管理特性来优化性能。

**注意：** 还有一点需要注意的就是，C++ 可以进行内存管理并直接操作内存，所以它会产生一系列问题，比如内存泄漏。

在 C++ 语言当中，需要格外注意内存泄漏的问题，必须完全考虑到对象的构造和析构。

由此引出了 C++ 的另一个特性——**RAII（资源获取即初始化）资源管理**。其原理是在构造时获取资源，在生命周期结束或析构的时候，对象的生命周期也随之结束。
- 构造即初始化
- 析构即关闭，作用域结束就会析构，**可以显式提供close（但是目的是避免手动关闭才会安全的原则）**。

零成本抽象：
C++ 具有零成本抽象（Zero-cost Abstractions）的特性。

什么是零成本抽象？

零成本抽象指的是对于 C++ 的模板或者内联函数，它们会在编译期间完全消失，被替换为运行内联或者模版后的结果。

相比之下，Python 会在运行时寻找相应的函数来执行。而 C++ 会将**使用内联函数或模板转换成确定的内容**，从而实现没有额外抽象成本的性能表现。


#### 还存在一个迭代器抽象
1. 对其抽象会在编译时将其优化成一个类似于**手写索引循环一样的机器码**，便于机器识别



#### 还存在一个编译期多态（CRTP）

是在编译阶段就已经将函数或变量的行为进行确定。

常规的多态行为（如虚函数多态）是在运行时确认行为，需要在运行的时候去查找虚函数表（vtable），这样会产生额外的开销。


#### 编译时计算等（constexpr）

在编译时将可以带有 constexpr关键字的东西进行计算，constexpr可以修饰一个函数。在编译的时候就会将其进行计算出来。

#### c++一定要注意值传递或者引用传递：

两者在不同的使用方式上面可能都会不同（比如在智能指针上面）

	两种方式的传递完全不同，值传递会拷贝，而引用不会拷贝，是别名，还是同一个变量。

**引用是别名，也不是指针变量，就是一个对象的别名，就是你引用的那个变量（就是给你的这个对象重新起了一个名字，其他都相同），一模一样，---因为其不需要解引用进行使用** 。
